# 생기부 점검 허브

NEIS 학교생활기록부(세부능력 및 특기사항·창의적 체험활동) 작성 규칙을 자동으로 점검하는 웹 도구입니다. 교과세특과 창체를 각각 점검하는 두 개의 도구를 하나의 허브에서 제공합니다.

- 랜딩: `/` — 도구 소개
- 교과세특 점검기: `/subject`
- 창체(동아리·진로·자율·행특) 점검기: `/career`

## 핵심 기능

- **과목별 구분**: NEIS 인쇄용 엑셀(학급 단위 다과목 파일)을 업로드하면 페이지 분할로 끊긴 과목 표기를 자동으로 복원해 과목별 섹션으로 정리합니다. 과목별 보기와 학생별 모아보기를 토글로 전환할 수 있습니다.
- **결과 화면에서 즉시 다운로드**: 점검 결과 화면 상단에 다운로드 3종이 항상 떠 있습니다.
  1. 전체 요약 xlsx (과목·이슈 상세 포함)
  2. 과목별 분리 xlsx (과목마다 개별 다운로드 + 전체 zip)
  3. 이슈 목록 클립보드 복사
- **100% 클라이언트 사이드**: 업로드한 학생 데이터는 브라우저 밖으로 나가지 않습니다. 서버로 전송·저장되지 않으며, 점검 상태는 각자의 브라우저 localStorage에만 남습니다.

## 사용법

1. NEIS에서 학급별 세특(교과세특 또는 창체) 엑셀을 내려받습니다.
2. 해당 도구(`/subject` 또는 `/career`) 페이지에 파일을 올립니다 — 드래그앤드롭 또는 파일 선택, 여러 파일 동시 업로드도 가능합니다.
3. 점검 결과 화면에서 과목별/학생별 보기를 전환하며 하이라이트된 이슈를 확인하고, 필요한 형식으로 바로 내려받습니다.

## 로컬 실행

빌드 과정이 없는 정적 사이트입니다. 프로젝트 루트에서 정적 파일 서버만 띄우면 됩니다.

```bash
python -m http.server 8000
```

이후 브라우저에서 `http://localhost:8000` 접속.

## 테스트

```bash
node tests/golden-parse.mjs
node tests/rules-parity.mjs
```

- `golden-parse.mjs`: NEIS 인쇄 덤프형 엑셀 파서가 표본 파일을 정확히 파싱하는지(명명행 수·과목 수·잘린 과목명 보정) 확인하는 골든 테스트입니다.
- `rules-parity.mjs`: 교과세특 규칙(`assets/js/rules-subject.js`)이 규칙 SSOT(`rule_scan.py`) 판정과 일치하는지 확인합니다.

> 표본 xlsx(`2026 1-5/1-6 과세특.xlsx` 등)는 실제 학생 개인정보를 포함하고 있어 이 저장소에는 포함되어 있지 않습니다. 위 두 테스트는 로컬에 표본 파일이 있을 때만 의미가 있는 **로컬 전용 게이트**이며, 표본이 없는 환경(CI 등)에서는 건너뜁니다.

## 배포

Vercel에 정적 사이트로 배포합니다. `vercel.json`의 `cleanUrls` 설정 덕분에 `/subject`, `/career`처럼 확장자 없는 경로로도 접근할 수 있습니다.

```bash
vercel deploy --prod
```

## 규칙 출처

점검 규칙은 학교생활기록부 기재요령을 기반으로 합니다. 교과세특 규칙은 단일 진실 원천(SSOT)인 `rule_scan.py`와 동기화되어 있으며, 규칙이 바뀌면 그 파일을 기준으로 이 저장소의 `assets/js/rules-subject.js`를 다시 맞춥니다. 동기화 과정에서 확인된 차이는 `docs/rules-sync-diff.md`에 기록되어 있습니다. 창체 규칙은 기존 점검기의 로직을 그대로 유지합니다.
